# WouldYouPay.io — Brand & Style Guide

> A comprehensive guide for the visual identity and design system of WouldYouPay.io

---

## 1. Brand Foundation

### 1.1 Brand Essence

**Mission:** Help builders validate ideas before they build

**Vision:** Become the first step in every builder's journey

**Values:**
- **Speed** — From idea to validation in minutes, not months
- **Honesty** — Real data, not vanity metrics
- **Community** — Builders helping builders
- **Craft** — Quality over quantity

### 1.2 Brand Personality

| Trait | Expression |
|-------|------------|
| **Technical** | We speak builder. No marketing fluff. |
| **Confident** | We know what we're for. Clear, direct. |
| **Playful** | Serious about building, not about ourselves |
| **Authentic** | Made by builders, for builders |
| **Distinctive** | We don't look like every other SaaS |

### 1.3 Taglines

**Primary:**
> "Validate before you build"

**Alternatives:**
- "Would they pay? Find out first."
- "Stop building things nobody wants"
- "The first step for every builder"
- "Test the idea, not your patience"

## 2. Logo & Wordmark

### 2.1 Direction

The logo should feel:
- Technical / builder-oriented
- Slightly retro (pixel/bitmap influence)
- Distinctive and memorable
- Works at small sizes (favicon)

**Concept Options:**

**A) Pixel Wordmark**
- "WouldYouPay" in custom pixel/bitmap typography
- Question mark as a distinctive element
- Monochrome primary, accent colour optional

**B) Abbreviated Mark**
- "WYP" or "WYP?" as a compact mark
- Pixel grid construction
- Works as favicon and app icon

**C) Symbol + Wordmark**
- Abstract symbol representing validation/testing
- Could be: checkbox, signal bars, thumbs up, coin
- Paired with clean wordmark

## 4. Typography

### 4.1 Type Philosophy

**Headlines:** Distinctive pixel/bitmap aesthetic — signals builder culture
**Body:** Clean, readable, modern — respects the content

### 4.2 Font Stack

**Display / Headlines:**
```
Primary:    "Departure Mono" or "Silkscreen" or "Press Start 2P"
Fallback:   "VT323", "IBM Plex Mono", monospace

Use for: H1, H2, hero text, brand moments
```

**Recommended pixel fonts to evaluate:**
- **Departure Mono** — Modern pixel aesthetic, excellent readability
- **Silkscreen** — Classic, clean pixel font
- **Press Start 2P** — Retro gaming feel (use sparingly)
- **Pixel Operator** — Versatile, multiple weights
- **VT323** — Terminal/computing aesthetic

**Body / UI:**
```
Primary:    "Inter" or "IBM Plex Sans"
Fallback:   -apple-system, BlinkMacSystemFont, sans-serif

Use for: Body text, UI elements, forms, navigation
```

### 4.3 Type Scale

```
Display XL:    48px / 56px line-height  (Hero headlines)
Display:       36px / 44px line-height  (Page titles)
H1:            28px / 36px line-height  (Section headers)
H2:            22px / 30px line-height  (Subsections)
H3:            18px / 26px line-height  (Card titles)
Body:          16px / 24px line-height  (Default text)
Small:         14px / 20px line-height  (Captions, meta)
Micro:         12px / 16px line-height  (Labels, badges)
```

## 5. Imagery & Illustration

### 5.1 Photography

**Generally avoid stock photography.** When images are needed:

- Real screenshots of the product
- User-generated content (actual landing pages)
- Abstract/geometric patterns
- Code snippets or terminal screenshots

### 5.2 Illustration Style

Inspired by Design Beyond Barriers:

**Characteristics:**
- Hand-drawn / sketch aesthetic
- Black line work on white
- Imperfect, human feel
- Technical subjects (computers, code, building)
- Occasional accent colour highlights

**Subjects:**
- Builders at work (coding, thinking, shipping)
- Abstract representations of ideas/validation
- Simple icons and diagrams
- Pixel art characters (optional, for personality)

### 5.3 Iconography

**Style:** Outlined, 2px stroke, rounded corners
**Grid:** 24x24px base, scale to 16, 20, 32, 48
**Colour:** Ink Black default, accent for emphasis

## 6. UI Components

### 6.1 Buttons

**Primary (CTA):**
```css
background: #FFE600;
color: #0D0D0D;
padding: 12px 24px;
border: 2px solid #0D0D0D;
font-family: "Departure Mono", monospace;
font-weight: bold;
text-transform: uppercase;
box-shadow: 4px 4px 0 #0D0D0D;

/* Hover */
transform: translate(2px, 2px);
box-shadow: 2px 2px 0 #0D0D0D;
```

**Secondary:**
```css
background: #0D0D0D;
color: #FAFAFA;
/* Same structure as primary */
```

**Ghost:**
```css
background: transparent;
color: #0D0D0D;
border: 2px solid #0D0D0D;
/* No shadow */
```

### 6.2 Cards

```css
background: #FAFAFA;
border: 2px solid #0D0D0D;
padding: 24px;
box-shadow: 6px 6px 0 #0D0D0D;

/* Hover (optional) */
transform: translate(-2px, -2px);
box-shadow: 8px 8px 0 #0D0D0D;
```

### 6.3 Form Inputs

```css
background: #FAFAFA;
border: 2px solid #0D0D0D;
padding: 12px 16px;
font-family: "Inter", sans-serif;

/* Focus */
outline: none;
box-shadow: 0 0 0 4px rgba(255, 230, 0, 0.5);
```

### 6.4 Navigation

- Minimal, stays out of the way
- Pixel font for logo/brand
- Clean sans-serif for nav items
- High contrast (black on white)

## 7. Layout Principles

### 7.1 Grid

**Desktop:** 12-column grid, 1200px max-width, 24px gutters
**Tablet:** 8-column grid, 16px gutters  
**Mobile:** 4-column grid, 16px gutters

### 7.2 Spacing Scale

```
4px   — Micro (icon padding, tight groups)
8px   — Small (related elements)
16px  — Medium (default spacing)
24px  — Large (section padding)
32px  — XL (major sections)
48px  — XXL (page sections)
64px  — Hero (above the fold)
```

### 7.3 Layout Patterns

**Hero Section:**
- Pixel font headline, centered or left-aligned
- Single clear CTA
- Generous whitespace
- Optional: subtle geometric pattern or illustration

**Feature Grid:**
- 2-3 column cards
- Consistent shadow depth
- Icon + headline + short description

**Social Proof:**
- Simple metrics or testimonials
- Understated design
- Real numbers, not fluff

## 8. Voice & Tone

### 8.1 Writing Principles

| Principle | Example |
|-----------|---------|
| **Direct** | "Test your idea" not "Begin your validation journey" |
| **Confident** | "Know before you build" not "Maybe find out if..." |
| **Technical** | Use builder vocabulary — ship, deploy, validate |
| **Concise** | Every word earns its place |
| **Honest** | No hype, no "revolutionary", no "game-changing" |

### 8.2 Do / Don't

**Do:**
- "Validate in 60 seconds"
- "Would they pay? Find out."
- "Stop building things nobody wants"
- "Made for builders"

**Don't:**
- "Revolutionary validation platform"
- "Synergize your ideation workflow"
- "Best-in-class solution"
- "It's like Uber for ideas"

### 8.3 Microcopy

**CTA buttons:**
- "Test This Idea"
- "Get Started"
- "Create Landing Page"
- "See Results"

**Empty states:**
- "No ideas yet. Ready to test one?"
- "Your first validation is free."

**Success:**
- "Page live. Start sharing."
- "12 people would pay. Nice."

**Errors:**
- "Something broke. Try again?"
- "That URL's taken. Pick another."

## 9. Motion & Interaction

### 9.1 Animation Principles

- **Purposeful** — Animation serves function, not decoration
- **Quick** — 150-300ms for most transitions
- **Subtle** — Enhance, don't distract
- **Consistent** — Same easing throughout

### 9.2 Micro-interactions

**Button hover:** Slight lift (translate + shadow)
**Card hover:** Elevation change
**Form focus:** Yellow glow ring
**Loading:** Simple pixel-style spinner or progress bar

### 9.3 Page Transitions

Keep simple. Fade or instant. No elaborate animations.
